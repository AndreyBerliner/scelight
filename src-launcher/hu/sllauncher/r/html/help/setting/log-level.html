The Log Level setting controls how detailed the log files managed by ${appName} should be.

Primary goal of log files is to give help later for bug and error analysis if something goes wrong in ${appName}.
Log files can be browsed and filtered inside ${appName}, but usually you don't need to do that. They are mainly needed if you encounter an error or a misbehaving, and you want to report a bug to the ${appName} author. In a bug report, you should always include your log files.

${error} log level entries indicate that something is not right or not how / what it should be; at least not under normal circumstances.

${warning} log level entries indicate potential problems or malfunctioning.

${info} log level entries are human readable and are intended for the user.

${debug} log level entries contain useful info about what is happening in the background.

The ${trace} log level also includes internal variables and program state useful only when analyzing anomalies.

Generally speaking the log level of ${debug} is recommended and is sufficient.
