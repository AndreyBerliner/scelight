Beans describing the modules.

The role of the different module beans is the following:

  • {@link hu.sllauncher.bean.module.ModulesBean ModulesBean}: acquired from the Application Operator, contains the latest versions of different modules.
  • {@link hu.sllauncher.bean.module.ModuleBean ModuleBean}: acquired from the Application Operator, describes an updateable module (both internal and external).
  • {@link hu.sllauncher.bean.module.ExtModRefBean ExtModRefBean}: acquired from the Application Operator, it is a reference to an official external module (to its {@link hu.sllauncher.bean.module.ModuleBean ModuleBean}).
  • {@link hu.sllauncher.bean.module.ExtModManifestBean ExtModManifestBean}: it is part of an external module, contains additional details and meta info about the external module (like its main class.)

ModuleBean is used to install/update/validate a module (either internal or external), so it has to contain info about its content to be able to validate it if the module is already present/installed.

ExtModRefBean must contain enough info for the user to be able to decide whether he/she wants to install it.

When there is no internet connection, only {@link hu.sllauncher.bean.module.ExtModManifestBean ExtModManifestBean} is accessible (from the local file system), so it has to contain all necessary info for the user and for the {@link hu.scelight.service.extmods.ExtModManager ExtModManager} to load and start it.
