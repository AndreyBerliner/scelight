APM stands for Actions Per Minute, it is a measurement of micro skills of players.
APM tells how fast players are during games. Numerically it shows how many actions / commands are issued by the player in 1 minute. Higher APM means more activity.

APM calculation is pretty simple: one just have to divide the number of actions with the number of minutes where the actions spread in time. For example 200 actions in 1 minute results in 200 APM.

The following actions are included in APM calculation:
  • Commands (Cmd events)
  • Selections (SelectionDelta events)
  • Control group commands (ControlGroupUpdate events)
APM calculation does NOT include:
  • Screen movements (CameraUpdate events)
The APM chart visualizes the current APM value of players at each time of the game, averaging by the selected granularity time value.

Note: The static APM of a player calculated for a game excludes the first 80 seconds (can be changed) and is only counted until the player's last command (Cmd event).
