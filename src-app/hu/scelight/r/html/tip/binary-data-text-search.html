Texts found in binary data are usually UTF-8 encoded.

Concept: This text search first converts the searched text to bytes using the same encoding (UTF-8), and this byte sequence will be searched. Therefore it will find the searched text in the binary data even though looking at the displayed characters you might see a different text (because it is not UTF-8 decoded but rather a simple one-byte ⇒ one-char projection ).

Additionally to the concept, the search is case insenstive. The searched text is first lower-cased before converting to byte sequence. If a compared byte is in the range of 'A'..'Z' , the corresponding lower-cased byte from the range of 'a'..'z' will also be accepted.
